METHOD & PATH DESCRIPTION

GET
api/person/complete Returns all persons with name and hobbies as a list of json objects
api/person/complete/{id} Returns the person with the given {id} as a json object (Remeber in our DB only every second id is a person, the rest is companies).
api/person/contactinfo Returns all persons with name and contact information, as a list of json objects
api/person/contactinfo/{id} Returns the person, with the given {id}, and it's contact information, as a json object

POST
api/person Creates a new person, if provided with a valid json object. A person may be created with firstName, lastName and hobbyList like this: { "firstName": "Peter", "lastName": "Lefort", "hobbyList": [ { "HobbyName": "Soccer" }, { "HobbyName": "Ice skating" } ] }. Or it may simply be created with only name like this: { "firstName": "Peter", "lastName": "Lefort" }

PUT
api/person/{id} Edits a person if provided with a valid person json object and an existing {id}. May use the same person constructors as given in the POST section above

DELETE
api/person/{id} Deletes the person with the given {id}.
